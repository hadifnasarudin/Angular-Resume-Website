Overview

```

Testing an existing web application through automated user workflows.

The Student Management System used in this project was an existing open-source application developed by Amir Hamza and obtained from GitHub. The system itself was not developed by me.

My contribution focused on software quality assurance. I studied the system's existing functions, identified important user workflows, developed automated test cases using Cypress, and generated structured test execution reports using Allure Report.

Project Objective

```

Evaluating whether the existing system functions correctly and consistently

The main objective of this project was to test the existing Student Management System using automated end-to-end testing. Cypress was used to simulate user actions and verify whether important system functions produced the expected results.

Automated testing helped reduce repetitive manual testing, improve test consistency, and identify whether key workflows continued to operate correctly across multiple test executions.

```

Testing Scope

```

Validating important student management workflows.

  • Application page loading
  • User navigation
  • Student record creation
  • Student information validation
  • Student record updates
  • Student record deletion
  • Search and filtering functions
  • Form submission behavior

Testing Approach

Recreating user actions through automated test scenarios.

  1. Reviewed the existing system and identified the main functions available to users.
  2. Selected important workflows that required functional verification.
  3. Created Cypress test cases based on expected user actions and system outcomes.
  4. Added assertions to verify page content, form behavior, navigation, and system responses.
  5. Executed the tests and reviewed passed and failed test cases.
  6. Generated Allure reports to present the test execution results clearly.
```

Testing Features

```

Main testing activities completed during the project

End-to-End Testing

Automated Test Cases

Assertions

Used Cypress assertions to confirm whether elements, messages, records, and page states matched the expected outcomes.

Framework Learning

```

Practical experience in automated software testing.

This project improved my understanding of Cypress commands, selectors, assertions, test structure, asynchronous browser behavior, and automated end-to-end testing.

I also learned how Allure Report can transform raw test execution results into structured reports that are easier to review and present.

Challenges

Creating reliable tests for an existing system.

One of the main challenges was understanding a system developed by another person before creating suitable test cases. Some interface elements and system behaviors also required careful selector selection to ensure that the tests remained stable.

Another challenge was connecting Cypress test execution with Allure Report and ensuring that the generated reports displayed the required test information.

Project Outcome

A documented automated testing workflow.

The completed project demonstrated how an existing web application can be evaluated using automated testing. Cypress was used to verify important functions, while Allure Report provided clear documentation of the test execution outcomes.
