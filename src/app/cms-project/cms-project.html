Academic Project Case Study

Conference Management System

Web-Based Platform for Managing Conference Activities and Workflows

The Conference Management System is a web-based academic project developed to manage important conference processes such as paper submission, review management, conference setup, scheduling, registration, and payment.

Angular Spring Boot MySQL GitHub
```

Overview

```

Simplifying conference management through an organised digital platform.

The system was designed to support different conference activities within one platform. It allows administrators, reviewers, and other users to manage papers, reviews, conference information, schedules, registration records, and payments more efficiently.

By organising these functions into separate modules, the system provides a clearer workflow and reduces the need to manage conference activities manually across different tools.

Technology Stack

```

Built using frontend, backend, database, and version-control technologies

Angular

Used to develop the frontend interface, navigation, reusable page components, and user interaction within the web application.

Spring Boot

Used to support backend application logic, API development, and communication between the frontend and database.

MySQL

Used to store and manage conference data such as papers, reviews, sessions, registrations, and payment records.

GitHub

Used for version control, source code management, project backup, and collaboration throughout development.

Core Modules

```

Supporting the main processes involved in conference management.

  • Paper Management
  • Review Management
  • Conference Setup
  • Scheduling
  • Registration Management
  • Payment Management

User Roles

Different interfaces and functions for administrators and reviewers.

Administrator

  • View and manage all submitted papers
  • Manage conferences, events, and tracks
  • Manage sessions, rooms, and schedules
  • Manage registration and payment records
  • Monitor conference-related activities

Reviewer

  • Access assigned paper reviews
  • View available papers
  • Submit review scores and feedback
  • Monitor review progress
  • View conference session information

Project Outcome

```

A centralised platform for managing conference activities.

The project demonstrated how a web application can combine multiple conference processes into one organised platform. It provided practical experience in frontend development, backend services, database integration, and user-role management.

Challenges

Managing multiple modules and user workflows.

One of the main challenges was ensuring that each module remained organised while still working as part of the same system. The project also required careful planning of navigation, database relationships, user roles, and communication between the frontend and backend.

Future Improvements

Opportunities to improve the system further.

  • Add stronger role-based access control.
  • Improve dashboard analytics and conference reporting.
  • Add automated email updates for important activities.
  • Improve payment verification and transaction tracking.
  • Add real-time schedule updates for conference participants.
  • Improve mobile responsiveness and accessibility.
